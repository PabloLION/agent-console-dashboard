# Story: Usage API Client

**Story ID:** S011.04 **Epic:**
[E011 - Claude Usage Crate](../epic/E011-claude-usage-crate.md) **Status:** Done
**Priority:** P0 **Estimated Points:** 3

## Description

As a developer, I want the `claude-usage` crate to call the Anthropic OAuth usage API, So that I can retrieve current usage data with a single function call.

## Context

Anthropic provides an OAuth usage API endpoint (`https://api.anthropic.com/api/oauth/usage`) that returns current utilization percentages for 5-hour and 7-day windows. This story implements the HTTP client that calls this API using credentials retrieved from platform-specific storage.

The client must handle common HTTP errors (401, 429, 5xx) gracefully and return structured error types.

## Implementation Details

### Technical Approach

1. Use `reqwest` with blocking client (async optional via feature flag)
2. Call `GET https://api.anthropic.com/api/oauth/usage`
3. Include required headers:
   - `Authorization: Bearer <token>`
   - `anthropic-beta: oauth-2025-04-20`
4. Handle HTTP status codes appropriately
5. Return raw JSON string for parsing in S011.05
6. Token is passed in, used once, never stored

### API Endpoint

```text
GET https://api.anthropic.com/api/oauth/usage
Authorization: Bearer <token>
anthropic-beta: oauth-2025-04-20
```

## Acceptance Criteria

- [ ] Given valid token, when `fetch_usage_raw()` is called, then API response JSON is returned
- [ ] Given invalid/expired token (API returns 401), then `Unauthorized` error is returned
- [ ] Given rate limit hit (API returns 429), then `RateLimited` error is returned with retry-after
- [ ] Given server error (API returns 5xx), then `ServerError` is returned
- [ ] Given network failure, when request fails, then `NetworkError` is returned
- [ ] Given successful call, when function returns, then token is not retained

## Notes

### Implementation

```rust
// crates/claude-usage/src/client.rs

use reqwest::blocking::Client;

const USAGE_API_URL: &str = "https://api.anthropic.com/api/oauth/usage";
const BETA_HEADER: &str = "oauth-2025-04-20";

pub fn fetch_usage_raw(token: &str) -> Result<String, ApiError> {
    let client = Client::new();

    let response = client
        .get(USAGE_API_URL)
        .header("Authorization", format!("Bearer {}", token))
        .header("anthropic-beta", BETA_HEADER)
        .send()
        .map_err(|e| ApiError::Network(e.to_string()))?;

    match response.status().as_u16() {
        200 => response.text().map_err(|e| ApiError::Network(e.to_string())),
        401 => Err(ApiError::Unauthorized),
        429 => {
            let retry_after = response
                .headers()
                .get("retry-after")
                .and_then(|v| v.to_str().ok())
                .map(String::from);
            Err(ApiError::RateLimited { retry_after })
        }
        500..=599 => Err(ApiError::Server(response.status().as_u16())),
        code => Err(ApiError::Unexpected(code)),
    }
}
```

### Error Types

```rust
#[derive(Debug, thiserror::Error)]
pub enum ApiError {
    #[error("Network error: {0}")]
    Network(String),

    #[error("Unauthorized. Token expired or invalid. Run `claude` to re-login.")]
    Unauthorized,

    #[error("Rate limited. Retry after: {retry_after:?}")]
    RateLimited { retry_after: Option<String> },

    #[error("Server error: HTTP {0}")]
    Server(u16),

    #[error("Unexpected status code: {0}")]
    Unexpected(u16),
}
```
